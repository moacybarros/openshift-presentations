Team Collaboration

Budgeted time: 3%

Now that you have a local, OpenShift-compatible source code repository, sharing a clone of this codebase on GitHub will allow you to easily collaborate with the rest of your team:

  1. Sign up for a free GitHub account
  2. Create an empty GitHub repo with a matching project name (no README or other starter files)
  3. Push your local OpenShift project to GitHub (substituting your PROJECT_NAME, and GITHUB_USERNAME where listed below):
cd PROJECT_NAME
git remote add github git@github.com:GITHUB_USERNAME/PROJECT_NAME.git
git push github master

Note: New team members will be able to skip step two above if they are added as a "collaborator" on another team-member's GitHub repository.

Additional team members will now be able to spin up their own demo servers by completing these two extra steps:

  • Spin up another demo environment based on the team's public source code:
rhc app create YOUR_APP_NAME YOUR_DEVELOPMENT_LANGUAGE --from-code=http://github.com/PROJECT_USERNAME/REPO_NAME.git
  • Link your local repo source to the team's shared GitHub repo to share code with other members:
cd PROJECT_NAME
git remote add github git@github.com:GITHUB_USERNAME/PROJECT_NAME.git

After committing your changes locally, or grabbing a fresh code update from your team members. You'll be able to update your own demo site by typing:

git push

Next, share your progress with your team by publishing your updates to GitHub:

git push github master

Team members can access the most recently shared code by pulling down updates from Github:

git pull github
