Why Node?

  • non-blocking event-driven applications
  • native content streams (realtime)
  • the event loop (a single thread)
  • it's good for applications that are IO bound rather than CPU bound
  • event emitters, callbacks, and contexts
  • compare to sinatra, flask, bottle, jaxRS, slimPHP, etc...

Node.js basics


        var http = require('http');
http.createServer(function (req, res) {
  res.writeHead(200, {'Content-Type': 'text/plain'});
  res.end('Hello World\n');
}).listen(1337, '127.0.0.1');
console.log('Server running at http://127.0.0.1:1337/');

native sockets


var net = require('net');
var server = net.createServer(function (socket) {
  socket.write('Echo server\r\n');
  socket.pipe(socket);
});
server.listen(1337, '127.0.0.1');
